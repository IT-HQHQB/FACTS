3.1 Authentication and Role-Based Access Control (RBAC)

3.11 Description

The system utilizes a comprehensive, stateless JSON Web Token (JWT) architecture to manage authentication and authorization. It employs a highly granular Role-Based Access Control (RBAC) model where permissions are resolved dynamically not just by the user's assigned role, but intersected strictly against their explicitly configured geographical scope constraints (jamiat_id and jamaat_id boundaries). The application prevents any traversal of Case data that falls outside a user's geographical assignment unless they possess global Super Admin overrides.
